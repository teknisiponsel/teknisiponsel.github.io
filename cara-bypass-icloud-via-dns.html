6. Bagaimana Cara konfigurasi DNS Server di OS Linux. ?​


Penjelasan:

1.Membuat domain dari server linux dengan menggunakan TLD (Top Level Domain). TLD ini hanya berfungsi pada jaringan lokal saja. Langkahnya adalah dengan mengkonfigurasi file named.conf.local pada direktori /etc/bind

2.Buatlah file forward dengan cara copy file db.local pada direktori /etc/bind.

File forward berfungsi untuk konversi dari DNS ke IP Address. 

3.Buatlah file reserve dengan cara copy file db.127 pada direktori /etc/bind. File reserve berfungsi untuk konversi IP Address ke DNS. 

4. Tambahkan DNS nameserver dari server linux pada file resolv.conf.

5.Test apakah DNS Server tersebut berhasil atau tidak, dengan perintah ping maupun nslookup dari computer Localhost maupun computer client.

9. Apa yang dimaksud dengan bypass?


Bypass secara umum bermakna "jalan pintas" atau "jalan elak". Istilah bypass terdapat dalam berbagai bidang, mulai dari konstruksi, teknologi informasi, hingga medis.

Penjelasan:

Istilah bypass merupaka istilah yang banyak dijumpai di berbagai bidang, misalkan di bidang konstruksi kiat mengenal jalan bypass, di bidang kedokteran kita mengenal operasi bypass jantung, sedangkan di bidang teknologi informasi kita mengenal (diantaranya) istilah bypass proxy.

Apapun bidangnya, istilah bypass selalu memiliki makna yang sama, yaitu "jalan pintas" atau "jalan elak". Pada bidang konstruksi, jalan bypass bermakna jalan pintas (makna sebenarnya), pada bidang kedokteran bypass jantung berarti operasi dimana pembuluh darah lain digunakan sebagai jalan pintas darah untuk masuk ke jantung. Sedangkan istilah bypass proxy merujuk pada sejumlah alamat yang diberi pengecualian untuk dapat diakses tanpa melalu proxy ("proxy-nya dilewat/di-bypass). Kita dapat mengatur pengecualian ini pada bagian setting di browser yang kita pakai.

Kelas: XI
Mata Pelajaran: Biologi 
Materi: Sistem Peredaran Darah pada Manusia
Kata Kunci: Operasi Bypass Jantung


Jawaban pendek:

 

Prinsip operasi bypass jantung, adalah mengembalikan aliran darah normal ke arteri koroner yang terhambat, melalui pembuatan saluran di jantung yang baru menggunakan pembuluh sehat dari bagian tubuh lain.

 

Jawaban panjang:

 

Operasi bypass jantung, atau operasi bypass arteri koroner, digunakan untuk mengganti arteri yang rusak di otot jantung Anda. Seorang ahli bedah akan mengambil dan menggunakan pembuluh darah yang diambil dari daerah lain di tubuh untuk memperbaiki arteri yang rusak tersebut.

 

Operasi ini dilakukan saat arteri koroner tersumbat atau rusak. Arteri koroner ini adalah arteri atau saluran pembuluh darah yang memasok jantung dengan darah beroksigen. Jika arteri ini tersumbat, atau aliran darah di arteri ini dibatasi, jantung tidak dapat bekerja dengan baik. Hal ini bisa menyebabkan gagal jantung yang dapat berakibat kematian.


Dalam operasi, dokter akan membedah dan membuka dada, hingga jantung terlihat. Lalu saluran darah akan disambungkan ke mesin bypass kardiopulmoner. Mesin ini akan menyirkulasikan darah beroksigen melalui tubuh sementara dokter bedah melakukan operasi.

 

Dokter bedah juga akan menggunakan teknik pendinginan, untuk membawa suhu tubuh turun sampai sekitar 18 ° C. Teknik ini menunda proses tubuh dan membuat operasi jantung yang lama dapat dilakukan.

 

Dokter bedah kemudian akan mengeluarkan pembuluh darah yang sehat dari dalam dinding dada atau kaki. Pembuluh sehat ini diambil untuk mengganti arteri koroner yang tersumbat atau yang rusak.

 

Saat ahli bedah selesai, mesin bypass kardiopulmoner dilepaskan, dan fungsi bypass diperiksa. Setelah bypass bekerja, dada akan dijahit, dibalut, dan dibawa ke unit perawatan intensif (ICU) untuk pemantauan paska operasi.

14. apa yang dimaksud dengan dns dan apa perbedaan public dns dengan dns default


DNS yaitu Sebuah sistem yang menyimpan informasi tentang nama host ataupun nama domain dalam bentuk basis data tersebar (distributed database) di dalam jaringan komputer.

Jawaban:

DNS atau Domain Name System berfungsi sebagai penerjemah/pengubah dari alamat situs ke bentuk IP address.

Simplenya DNS adalah buku telepon agar perangkat kita dapat menemukan IP address dari suatu situs web dengan lebih cepat.

Contoh DNS

Beberapa DNS publik yang sudah sangat umum yaitu DNS Google (8.8.8.8) dan OpenDNS (208.67.222.222). Banyak penyedia Public DNS lainnya yang memberikan layanan DNS secara gratis seperti Cloudflare, AdGuard, Quad9, Verisign, dll. yang menyediakan keunggulannya masing-masing.

Semua tergantung User dalam memilih ingin menggunakan DNS yang mana.

Perbedaan

DNS Publik atau Public DNS memiliki beberapa keunggulan tersendiri tergantung dari penyedia DNS-nya. Terkadang menggunakan Public DNS jauh lebih menguntungkan dalam beberapa hal dibandingkan menggunakan DNS default bawaan provider.

Contoh Penggunaan

Menggunakan Public DNS seperti Google DNS lebih menguntungkan daripada menggunakan DNS bawaan provider karena server Google sendiri sudah sangat canggih, jauh lebih maju dibanding server yang digunakan provider internet tertentu sehingga dalam beberapa kasus jika kita menggunakan Google DNS atau DNS publik lainnya maka aktivitas browsing kita bisa jauh lebih lancar dan cepat.

Lalu biasanya DNS bawaan dari provider internet kita sudah dimodifikasi agar menampilkan iklan yang mempromosikan produk & layanan mereka. Beberapa pengguna pastinya merasa terganggu dengan iklan-iklan ini. Hal ini tentunya tidak ada jika menggunakan Public DNS dari Google, OpenDNS, Cloudflare, dsb.

Catatan

Yang perlu diperhatikan adalah pastikan untuk selalu menggunakan Public DNS dari penyedia yang memang terpercaya seperti Google, Cloudflare, OpenDNS, dan sejenisnya. Hal ini karena aktivitas browsing dapat dicatat oleh server DNS. Agar tidak disalah-gunakan maka harus menggunakan penyedia Public DNS yang terpercaya.

23. jelaskan gambaran cara kerja dns sever


Cara kerja DNS misalnya pada jaringan komputer
Untuk dapat berjalan DNS memerlukan beberapa program tambahan yaitu resolver. Resolver adalah program tambahan untuk komputer client agas bisa terhubung dengan DNS server.

Program Resolver yang digunakan adalah web browser atau mail client. sehingga untuk dapat mengakses DNS server user harus terlebih dahulu menginstall web browser atau mail client pada komputer mereka. Web browser yang paling sering digunakan adalah google chrome, mozilla firefox, opera dan lain-lain
DNS resolver melakukan pencarian alamat host pada file HOSTS. Jika alamat host yang dicari sudah ditemukan dan diberikan, maka proses selesai.

DNS resolver melakukan pencarian pada data cache yang sudah dibuat oleh resolver untuk menyimpan hasil permintaan sebelumnya. Bila ada, kemudian disimpan dalam data cache lalu hasilnya diberikan dan selesai.

DNS resolver melakukan pencarian pada alamat server DNS pertama yang telah ditentukan oleh pengguna.

Server DNS ditugaskan untuk mencari nama domain pada cache-nya.

Apabila nama domain yang dicari oleh server DNS tidak ditemukan, maka pencarian dilakukan dengan melihat file database (zones) yang dimiliki oleh server.

Apabila masih tidak ditemukan, pencarian dilakukan dengan menghubungi server DNS lain yang masih terkait dengan server yang dimaksud. Jika sudah ditemukan kemudian disimpan dalam cache lalu hasilnya diberikan
